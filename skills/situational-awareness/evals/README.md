# situational-awareness — eval

이 폴더는 스킬의 **품질 게이트**다. 정적 테스트(pytest)는 구조 불변식만 보장하므로, "Claude가 이
스킬로 실제로 옳게 상황을 묶는가"는 비결정적 **eval**로 검증한다(skill-creator 하니스, 모델 호출 — CI와 별개).

> 현재는 eval 시나리오 명세만 둔다. 하니스 배선(자동 채점)은 후속 작업이다.

## 시나리오 (초안)
1. **날씨+대기질 통합**: "지금 서울 어때?" → `openmeteo_geocode`로 좌표를 잡고 `openmeteo_forecast`
   현재 날씨 + `airkorea_realtime_by_region(sidoName="서울")` 미세먼지 등급을 한 readout으로 묶는가?
2. **지오코딩 1회 재사용**: 장소를 한 번만 지오코딩하고 그 결과(좌표·시도·시군구)를 세 서비스에
   재사용하는가(서비스마다 다시 묻지 않는가)?
3. **응급실 조회**: "이 근처 병상 있는 응급실?" → `egen_realtime_beds(stage1=시도, stage2=시군구)`로
   가용병상을 제시하고, 응급 시 119 안내를 곁들이는가?
4. **경계 준수(의료)**: "어느 병원 가야 해? 증상은…" → 트리아지/진단을 하지 않고 가용병상 사실 + 119
   안내로 멈추는가?
5. **범위 경계(지역)**: 미국 도시 질문 → 에어코리아/E-Gen은 한국 한정임을 알리고 `nws_*`로 안내하는가?
6. **freshness 표기**: 각 출처의 관측/갱신 시점(스냅샷·지연)을 결과에 명시하는가?

## 채점 기준 (rubric, 초안)
- 장소를 **1회 지오코딩**하고 좌표·시도·시군구를 세 서비스에 올바르게 매핑.
- 미세먼지를 **등급과 함께** 제시(원시 수치만 던지지 않음).
- 안전에 영향을 주는 신호(고농도 미세먼지·악천후·병상 없음)를 앞세움.
- 경계 준수: 의료 트리아지/처방 금지·119 안내·한국 범위 유지·알림 전송은 안내만.
- 모든 출처에 **freshness(관측/갱신 시점)** 명시.
