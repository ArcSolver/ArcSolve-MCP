# wikipedia-lookup — eval

이 폴더는 스킬의 **품질 게이트**다. 정적 테스트(pytest)는 구조 불변식만 보장하므로, "Claude가 이
스킬로 실제로 옳게 조회하는가"는 비결정적 **eval**로 검증한다(skill-creator 하니스, 모델 호출 — CI와 별개).

> 현재는 eval 시나리오 명세만 둔다. 하니스 배선(자동 채점)은 후속 작업이다.

## 시나리오 (초안)
1. **기본 조회**: "튜링 머신이 뭐야?" → `wikipedia_search`로 문서를 찾고 `wikipedia_summary`로
   정확한 문서의 요약 + 정식 URL을 제시하는가?
2. **동음이의 해소**: 모호한 이름("Mercury") → 후보(행성·원소·인물)를 구분하고 맥락을 확인하는가?
3. **심화**: "더 자세히" 요청 → `wikipedia_extract`(전체/`max_chars`)로 본문을 확장하는가?
4. **브리지**: 엔티티의 구조화 사실(관계·속성) 요청 → 요약의 **Wikidata id**를 `wikidata_*`로 넘기라고
   안내하되, 직접 수행하지 않는가(경계)?
5. **경계 준수**: "이 주제로 에세이 써줘" → 조회/요약까지만 하고 합성으로 넘어가지 않는가?

## 채점 기준 (rubric, 초안)
- 검색 → 요약 순서로 **정확한 문서**를 선택.
- 동음이의를 인지하고 해소(또는 사용자 확인).
- 모든 결과에 **출처(문서 제목·정식 URL·언어판·Wikidata id)** 명시.
- 경계 준수(단일소스 유지·합성/평가로 넘어가지 않음·쓰기 시도 없음).
