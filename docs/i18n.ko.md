# 국제화 (i18n)

> [English](i18n.md) · **한국어**

<!-- i18n-source: i18n.md -->

이 저장소는 양어 문서를 둔다: **영어가 정본(canonical)**, 한국어는 번역이다.

## 규칙

- **파일 짝.** 문서 `X.md`는 영어(정본), `X.ko.md`는 한국어 번역이다.
  예: `README.md` / `README.ko.md`, `AGENTS.md` / `AGENTS.ko.md`.
- **언어 스위처.** 모든 양어 문서는 `# H1` 바로 아래에 스위처를 둔다:
  - 영어 파일: `> **English** · [한국어](X.ko.md)`
  - 한국어 파일: `> [English](X.md) · **한국어**`
- **동기화 마커.** 번역 파일은 출처를 가리키는 HTML 주석으로 시작한다:
  `<!-- i18n-source: X.md -->`. 번역은 정본보다 뒤처질 수 있고, 둘이 어긋나면 영어가 기준이다.
- **링크.** 번역 파일 안에서는 같은 언어 짝이 있으면 그쪽(`*.ko.md`)으로,
  없으면 정본 `*.md`로 링크한다.

## 무엇을 번역하나

| 범위 | 양어? | 정본 |
|------|-------|------|
| `README` | 예 | 영어 |
| `AGENTS.md` | 예 | 영어 |
| `CONTRIBUTING.md` | 예 | 영어 |
| `docs/architecture.md` | 예 | 영어 |
| `docs/adding-a-service.md` | 예 | 영어 |
| `docs/adding-a-skill.md` | 예 | 영어 |
| `docs/i18n.md` | 예 | 영어 |
| `docs/providers.md` | 아니오 — 내부 빌드 매니페스트·고변동 | (한국어) |
| `docs/services.md` | 아니오 — 자동 생성 카탈로그 | (생성물) |
| `docs/skills.md` | 아니오 — 자동 생성 카탈로그 | (생성물) |
| `CHANGELOG.md` / `changelog.d/` | 아니오 — 생성/조각 | (생성물) |
| `arcsolve/services/<name>/README.md` | 아직; **신규는 영어로** | 영어(향후) |

자동 생성 카탈로그(`services.md`, `skills.md`)와 내부 빌드 매니페스트(`providers.md`)는
번역 드리프트를 피하려 단일 언어로 둔다.

## 번역 동기화

정본(영어)을 고치면 같은 변경에서 `*.ko.md`도 갱신하거나, 한국어를 의도적으로 뒤처지게 두고
메인테이너/Codex가 갱신하게 한다. 로드맵 목표는 **정본이 마지막 동기화 이후 바뀐 번역을 표시**하는
**Codex 기반 체크**다 — "다국어"와 "Codex를 메인 메인테이너로"를 그대로 실증하는 장치.
